# Role playbook

## Product manager

Checks user value, scope, success metrics, dependencies, edge cases and whether a smaller release delivers the same outcome.

## UX designer

Checks information hierarchy, discoverability, long content, loading/empty/error states, mobile, keyboard flow and accidental actions.

## Growth/business

Checks acquisition/retention impact, event instrumentation, incentives, spam, manipulation and dark-pattern risk.

## Frontend architect

Checks component reuse, domain boundaries, server/client state, rendering cost, type safety and maintainability.

## Backend/API reviewer

Checks schemas, pagination, filtering, error model, idempotency, compatibility, rate limits and observability.

## Security/privacy

Checks trust boundaries, authentication, authorization, secrets, PII, wallet/signing, replay, injection and audit logs.

## SRE/operations

Checks health, timeouts, retry storms, feature flags, rollout, alerting, capacity and rollback.

## QA engineer

Builds a matrix of happy paths, boundaries, malformed data, concurrency, offline behavior and regressions.

## Accessibility reviewer

Checks semantics, labels, focus order, contrast, target size, zoom, reduced motion and screen-reader output.

## Performance reviewer

Checks bundle size, route splitting, fetch waterfalls, cache policy, re-renders and layout stability.

## Skeptical user

Challenges unclear probabilities, resolution sources, deadlines, fees, destructive actions and misleading confidence.
